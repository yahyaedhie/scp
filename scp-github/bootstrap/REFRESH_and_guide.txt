## SPF::REFRESH PACKET
## Paste inline at any point in long session to re-anchor.

SPF::REFRESH {
  protocol: "SCP v3.0",
  type: "mid-session-refresh",
  domain: "market",
  active_codes: ["[WAR]","[SH-C]","[SH-S]","[LIQ]","[CARRY]","[STACK]"],

    "shorthand+domain → fixed meaning always",
    "flag drift — never silently correct",

  instruction: "Re-confirm all active codes against anchors. Continue session."
}

## USAGE GUIDE

### LOADING STRATEGY

| Scenario                  | Load                        | Est. Tokens |
|---------------------------|-----------------------------|-------------|
| Single model, new session | T1 + T2 full pack           | ~600        |
| Same model, continuing    | T1 only                     | ~180        |
| Cross-model handoff       | T3 SPF packets only         | ~420        |
| Cold model, no context    | T1 + T3 embedded            | ~600        |
| Long session (>20 turns)  | Inject SPF::REFRESH         | ~80         |
| Compliance/audit mode     | T1 + T2 + T3 + SCV layer    | ~900        |

---

### FILE LOADING ORDER

```
1. T1_system_prompt.txt          → paste as system prompt
2. T2_anchor_pack_market.txt     → paste after system prompt OR inject on first use
3. T3_spf_packets_market.txt     → use for cross-model relay or cold start
4. SPF::REFRESH (inline above)   → paste mid-session as needed
```
